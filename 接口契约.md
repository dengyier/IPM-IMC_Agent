# 前后端接口契约（以前端 mock 为基准反推）

> 基准来源：`frontend/lib/data.ts` + 各页面组件内联 mock
> 原则：后端只返回**语义数据**（枚举/数值/结构化字段）；前端负责**样式映射**（CSS class、icon、tint、accent）与展示串拼接。
> 状态标记：✅ 已有后端端点 ｜ 🟡 端点在但字段需补 ｜ ❌ 需新建

---

## 0. 全局约定

- 状态枚举（后端返回英文，前端映射中文+样式）：
  - 资料处理：`uploaded｜processing｜processed｜kernel_built｜failed`
  - 审核：`pending｜approved｜rejected｜revise_required`
  - 报告：`generating｜generated｜approved｜failed`
- 前端展示映射示例（前端侧维护，**不进后端**）：
  ```ts
  const statusTone = { processed:"bg-emerald-50 text-emerald-600", processing:"bg-orange-50 text-orange-500", failed:"bg-rose-50 text-rose-500" }
  ```
- 响应是否包信封（`{success,data,...}`）：**待定**——前端现在直接消费裸对象，若包信封前端取数代码需全改。建议**不包**，或仅错误时包。

## 9. 异步任务（长任务统一机制）

实测建图 63 分钟、diagnose 数十秒；BackgroundTasks 同进程、重启即丢（本项目已踩坑）。

- 长任务接口返回 `{ task_id, status:"running" }`
- 轮询 `GET /api/tasks/{id}` → `{ status, progress, result, error }`
- **v1 即要求**：任务状态**落库**（复用 AgentRun）+ 脱离请求生命周期，不用纯内存。
- 需异步：`methodology process / build-kernel`、`expansion absorb`、`diagnosis diagnose`、`report regenerate`。

## 10. 后端缺口汇总（按 §14 闭环优先级排序）

1. ✅ **Dashboard**：`/api/dashboard/summary`、`/recent-reports`、`/recent-review-tasks` + 聚合 service
2. ✅ **节点子资源**：`nodes/{id}/edges`、`/versions`、`/expansions`；列表补 `edge_count`、`node_category` 分组
3. ✅ **异步任务**：`/api/tasks/{id}`（+列表）+ `tasks` 表落库 + 线程池脱离请求；process/build-kernel/diagnose 已改异步返回 `{task_id,status}`；前端 `tasksApi`+`pollTask`
4. ✅ **system health 明细** + `test-llm` / `test-vector-store`（+只读 `GET /api/system/settings` 掩码）；前端 `systemApi`
5. ✅ **扩展条目详情** `expansion/items/{id}`（内联来源 + 对齐节点）；**审核任务详情** `review/tasks/{id}`（内联完整条目详情）；前端 `expansionApi`/`reviewApi`
6. ✅ **报告** `regenerate`（异步，新 report_id）/ `DELETE`（204）；前端 `reportsApi`
7. （后置）统一信封、`/api/settings` 写、鉴权、用户体系（listTabs/people 字段依赖它）

## 11. 前端侧改造（原后端计划未覆盖，但「以前端为契约」必然包含）

- 新增 `lib/api.ts`（fetch 封装 + base url + 错误处理）
- 各页面从 `import 常量` 改为 `fetch + state/loading`
  - **全部 7 个页面已接入真实数据**（零 mock）：Dashboard、知识节点库、系统设置、诊断报告中心（list/detail/quality/regenerate/delete + 质量分 0~1→0~100）、人工审核台（tasks/task/decide + 状态分组/搜索/分页）、资料中心（两池合并 + 上传 multipart + process/build-kernel 异步轮询 + absorb）、商业画布诊断（9 宫格表单 → diagnose 异步 → 内联产出）、同学笔记进化（上传/粘贴 → absorb → 匹配节点 + 增量条目 → 跳审核）

- 新增 `lib/presentation.ts`：状态枚举→CSS class / icon / tint 映射
- `lib/data.ts` 降级为「兜底/Storybook 用」或删除
