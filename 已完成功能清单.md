## 三、Phase 1：方法论底座（核心知识图谱）

**目标**：把港大 IMC&IPM 课件解析为可检索、可推理的统一知识图谱。

### 功能项
- [x] 文档解析（PDF / PPTX，PyMuPDF / python-pptx）
- [x] 分页切块（chunk）+ 主题/章节标注
- [x] 可插拔向量化（本地确定性哈希 / OpenAI 兼容端点）
- [x] 向量库封装（Qdrant，带内存回退 + payload 分层过滤）
- [x] LLM 批量节点抽取（DeepSeek，支持分批 + 跨课件去重合并）
- [x] 节点落库（定义/核心原理/核心思维/决策逻辑/关键问题/常见错误/适用场景）
- [x] 关系边构建（8 种关系类型，4 信号加权）
- [x] 问题路由规则生成（10 类商业问题意图）

### 关键算法
- **边权公式**：`weight = 0.4·语义 + 0.3·共现 + 0.2·决策顺序 + 0.1·人工确认`
- **8 种关系**：prerequisite / supports / causes / constrains / validates / extends / contrasts / risk_trigger
- **10 类问题意图**：项目可行性 / 商业模式设计 / 客户定义 / 价值主张校验 / 收入模型校验 / 风险诊断 / 市场进入 / 品牌定位 / 组织执行 / 投资决策

### 对应文件
`methodology_kernel_service.py` · `knowledge_graph_service.py` · `problem_routing_service.py` · `document_parser.py` · `embeddings.py` · `vector_store.py` · `graphs/methodology_kernel_graph.py`

### API
- [x] `POST /api/methodology/upload` 上传课件
- [x] `POST /api/methodology/process` 解析+切块+向量化
- [x] `POST /api/methodology/build-kernel` 构建图谱

## 五、Phase 3：商业画布诊断

**目标**：基于商业模式画布提问，路由命中方法论 → 融合上下文 → 生成诊断 → 质量自检。

### 功能项
- [x] 问题路由（按关键词命中评分选最佳规则）
- [x] 上下文融合（节点 + 核心切块 + 已批准扩展 + 案例 + 时效）
- [x] 诊断生成（DeepSeek + 本地兜底，9 大画布模块）
- [x] 报告质量自检（7 维评分 + 核心内容泄漏检测）

### 关键算法
- **上下文融合**：`final = 0.45·节点 + 0.25·核心切块 + 0.15·已批准扩展 + 0.10·案例 + 0.05·时效`
- **质量自检**：`score = 0.20·画布完整性 + 0.20·方法论一致性 + 0.15·假设 + 0.15·风险 + 0.15·可执行性 + 0.10·证据 + 0.05·安全`，**通过阈值 0.70**
- **9 大画布模块**：客户细分 / 价值主张 / 渠道 / 客户关系 / 收入来源 / 核心资源 / 关键业务 / 重要伙伴 / 成本结构

### 对应文件
`context_fusion_service.py` · `diagnosis_service.py` · `report_quality_service.py` · `graphs/business_canvas_diagnosis_graph.py` · `models/diagnosis.py`

### API
- [x] `POST /api/diagnosis/diagnose` 发起诊断
- [x] `GET  /api/diagnosis/reports` 报告列表
- [x] `GET  /api/diagnosis/quality` 质量自检结果
