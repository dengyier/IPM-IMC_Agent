# 天机AI商业决策智能体 PC Web 设计稿 v0.4 说明

## 设计方向

本版设计围绕最新 PRD 的核心链路重构：

企业问题入口 -> 数据/资料沉淀 -> 知识图谱/RAG -> 多角色决策推演 -> 风险审计 -> 验证计划 -> 经营档案 -> 方法论复利。

设计目标不是做一个“报告后台”，而是把天机AI表现为一个持续工作的经营决策系统：用户从一个模糊问题进入，系统逐步形成证据、判断、风险、验证动作和经营档案。

## 关键变化

- 首页只保留“你现在最想验证什么？”作为核心入口。
- AI 经营访谈采用大模型产品式布局，同时保留天机AI的证据抽屉、知识节点引用、经验沉淀和项目诊断入口。
- 项目验证诊断强调九宫格画布、多角色推演、风险审计与验证计划，不再只是填写表单。
- 诊断报告中心突出“执行摘要、核心矛盾、风险矩阵、7天验证动作”。
- 经营档案以 Project 为核心，聚合对话、资料、报告、验证卡、复盘记录。
- 资料中心明确展示上传、解析、分块、向量化、抽取、审核的处理流程。
- 人工审核台支持一键通过/拒绝，并展示 AI 摘要、建议入库位置和来源追溯。
- 系统设置按 MVP 简化，仅保留用户名称、企业信息、企业类型、联系方式和退出登录。

## 实现建议

- 先按页面信息架构落地，不急于复刻所有视觉细节。
- 左侧导航和知识资产卡作为全局布局组件。
- AI 经营访谈、经营档案、报告中心优先对齐项目实体 `Project`。
- 所有“沉淀”动作都应进入审核流，未审核内容不要进入正式知识库。
